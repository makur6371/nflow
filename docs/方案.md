# nflow 项目开发方案

> **项目名称**：nflow（对标 iflow）
> **全称**：novel-flow（便于理解）
> **定位**：AI 小说写作 CLI 工具

## 一、项目背景

### 1.1 市场现状调研

#### 1.1.1 发展趋势
- **2024年7月**：AI只能辅助扩写润色
- **2024年底**：AI能直接写正文
- **2025年初**：AI能写大纲了
- AI能力呈现指数级增长，商业化案例已有月入百万的项目

#### 1.1.2 主流产品

**商业化产品（国内）**
- 蛙蛙写作：一站式AI小说写作助手，支持从构思到成文全过程
- FeelFish：多模态创作、多语言支持
- 笔灵AI：专注网文创作
- 星野写作助手：强于世界观构建

**开源项目（GitHub）**
- NovelGenerator (KazKozDev)：使用LLM agents创建完整小说
- 302_novel_writing (302ai)：支持手动编辑和AI生成

#### 1.1.3 技术特点
- Agent架构：多Agent协作模式
- 端到端工作流：大纲→人物设定→正文生成→润色修改
- 多语言支持：支持中文、英文等多语言创作
- 专业化细分：针对网文、剧本、小说等不同类型

### 1.2 现有问题与不足

#### 1.2.1 连贯性不足
- 长篇小说容易前后矛盾
- 人物性格不稳定
- 时间线混乱

#### 1.2.2 创意瓶颈
- 模式化写作，缺乏新意
- 剧情走向可预测
- 难以写出深刻的主题

#### 1.2.3 理解能力有限
- 对复杂情感、隐含信息把握不准
- 文化背景理解表面化
- 缺乏人生体验支撑

#### 1.2.4 需要大量人工干预
- 需要频繁调整提示词
- 大量编辑修改才能用
- 效率提升不如预期

### 1.3 现实定位

目前AI更适合：
- **辅助创作**：生成灵感、扩写段落、润色文字
- **快速产出**：内容农场、短篇内容
- **学习练手**：新手练习写作

真正的高质量小说创作，仍需要人类的深度参与。

## 二、iflow-cli 深度分析

### 2.1 iflow-cli 项目概览

**项目地址**：https://github.com/iflow-ai/iflow-cli

**项目描述**：
- iFlow CLI 是一个强大的 AI 助手，直接在终端运行
- 能够无缝分析代码仓库、执行编程任务、理解上下文需求
- 通过自动化处理从简单文件操作到复杂工作流程

**核心特性**：
- ✅ 免费AI模型（Kimi K2、Qwen3 Coder、DeepSeek v3 等）
- ✅ 灵活集成（保留现有开发工具，集成自动化系统）
- ✅ 自然语言交互（日常对话驱动 AI）
- ✅ 开放平台（一键安装 SubAgents 和 MCP）

**运行模式**（4种）：
1. **yolo**：模型拥有最大权限，可执行任何操作
2. **accepting edits**：模型仅有文件修改权限
3. **plan mode**：先规划后执行
4. **default**：模型无权限

### 2.3 iflow-cli 核心架构

#### 2.3.1 分层、事件驱动架构

**核心组件**：

1. **Tool System（工具系统）**
   - 工具的注册、发现和执行
   - 工具定义为继承 `BaseTool` 的类
   - 参数验证和权限控制

2. **Agent System（代理系统）**
   - 子代理编排（`general-purpose`、`plan-agent`、`explore-agent`）
   - 并行任务执行
   - 每个代理有独立的工具权限和 MCP 访问

3. **MCP Integration（MCP 集成）**
   - 原生支持 Model Context Protocol 服务器
   - 从外部进程动态发现工具

4. **Event System（事件系统）**
   - 组件间异步通信
   - 工具调用、代理生命周期、UI 更新

5. **Configuration & Auth（配置 & 认证）**
   - 集中式配置管理
   - 多认证提供商（iFlow 原生、OpenAI 兼容）

#### 2.3.2 UI 层架构

- **React-based UI**：使用 Ink 进行终端渲染
- **Command Parser**：处理 slash 命令、直接代理调用、自然语言查询
- **State Management**：React hooks 管理对话历史、工具调用状态、用户偏好

#### 2.3.3 安全与监控

- **Sandbox Environment**：可选的 Docker/Podman 隔离
- **Telemetry**：OpenTelemetry 指标、跟踪、日志
- **Git Integration**：自动检测仓库、生成提交消息、查看差异

## 三、项目核心理念

### 3.1 本质：项目改造

**nflow 是 iflow-cli 的小说化改造版本**
（全称：novel-flow，便于理解）

这不是从零开发，而是基于 iflow-cli 成熟架构的深度改造：

| iflow-cli | nflow |
|-----------|-------|
| 写代码 | 写小说 |
| 构架项目 | 框架小说 |
| 代码理解 | 文本理解 |
| 函数/模块 | 章节/段落 |
| 调试 | 审核 |
| 重构 | 润色 |

### 3.2 核心映射关系

```
iflow-cli 能力              →  nflow 能力
────────────────────────────────────────────
理解编程需求                →  理解创作意图
生成代码文件                →  生成小说章节
调用子agent处理不同任务     →  调用子代理处理不同创作环节
代码质量检查                →  文本质量审核
上下文管理                  →  小说记忆管理（人物、情节、设定）
迭代优化                    →  迭代创作
Tool System                 →  Writing Tool System
Agent System                →  Creation Agent System
MCP Integration             →  Writing Resource Integration
Event System                →  Creation Event System
```

### 3.3 两个核心目标

#### 目标一：把小说写作能力做大做强
- 不是简单的文本生成，而是深度的创作能力
- 支持长篇、复杂、高质量的小说创作
- 解决连贯性、创意、理解力三大痛点

#### 目标二：彻底小说化改造
- 术语映射：代码术语 → 小说术语
- 交互优化：符合创作者的使用习惯
- 工具适配：为小说创作定制的工具链

### 3.4 项目定位

- **基于 iflow-cli 架构**：复用成熟的技术栈和工作流
- **深度小说化**：不是简单改名，而是全方位改造
- **开源项目**：提供完整的可扩展框架
- **CLI工具**：命令行交互，高效便捷

#### 4.2.6 字数控制模块（新增）

**目标**：确保每章字数在合理范围内，避免忽高忽低

**字数标准**（基于番茄小说平台要求）：

| 平台要求 | novel-flow 设置 |
|---------|----------------|
| 每章建议：2050-2300字 | **字数范围：2000-2500字** |
| 利于有声小说 | **目标字数：2200字** |
| 发布范围：2000-6000字 | **允许误差：2000-2400字** |

**功能特性**：

1. **章节字数设定**
   - 创建项目时设定目标字数（默认2200字）
   - 每章独立字数配置（支持差异化）
   - 字数范围验证（最小2000字，最大3000字）

2. **实时字数监控**
   - 写作过程中实时统计字数
   - 字数进度可视化
   - 接近目标时提醒

3. **自动字数调整**
   - **字数不足（<2000字）**：自动扩展内容
     - 扩展环境描写
     - 增加心理描写
     - 补充对话内容
     - 添加细节刻画
   - **字数超标（>2400字）**：自动压缩内容
     - 压缩冗余描写
     - 合并相似段落
     - 删除重复信息
     - 精简对话

4. **字数报告生成**
   - 每章字数统计
   - 全书字数分布图
   - 字数达标率
   - 字数趋势分析

**技术实现**：

```typescript
// 字数控制工具
class WordCountController {
  // 字数配置
  config = {
    target: 2200,        // 目标字数
    min: 2000,           // 最小字数
    max: 2400,           // 最大字数（自动修复阈值）
    absoluteMax: 3000    // 绝对最大字数
  };

  // 实时统计字数
  countWords(text: string): number {
    return text.replace(/\s+/g, '').length;
  }

  // 检查字数是否达标
  checkWordCount(text: string): WordCountStatus {
    const count = this.countWords(text);
    if (count < this.config.min) {
      return { status: 'insufficient', count, shortage: this.config.min - count };
    } else if (count > this.config.max) {
      return { status: 'excess', count, excess: count - this.config.max };
    } else {
      return { status: 'optimal', count };
    }
  }

  // 自动扩展字数
  async expandContent(chapter: string, target: number): Promise<string> {
    const current = this.countWords(chapter);
    const needed = target - current;

    // 使用AI扩展内容
    const prompt = `
    当前章节字数：${current}字
    目标字数：${target}字
    需要增加：${needed}字

    请在保持原有内容和风格的基础上，自然地扩展以下章节，重点增加：
    - 环境描写细节
    - 人物心理活动
    - 动作描写
    - 感官体验

    章节：
    ${chapter}
    `;

    const expanded = await this.callAI(prompt);
    return expanded;
  }

  // 自动压缩字数
  async compressContent(chapter: string, target: number): Promise<string> {
    const current = this.countWords(chapter);
    const excess = current - target;

    // 使用AI压缩内容
    const prompt = `
    当前章节字数：${current}字
    目标字数：${target}字
    需要删除：${excess}字

    请在保持核心情节和风格的基础上，精简以下章节，重点：
    - 删除冗余描写
    - 合并相似内容
    - 删除重复信息
    - 精简对话

    章节：
    ${chapter}
    `;

    const compressed = await this.callAI(prompt);
    return compressed;
  }
}
```

**质量评分系统**：

```typescript
class QualityScorer {
  // 评分维度
  dimensions = {
    characterConsistency: 30,   // 人物一致性 30%
    plotLogic: 25,              // 情节逻辑 25%
    worldSetting: 20,           // 世界观设定 20%
    timeline: 15,               // 时间线 15%
    foreshadowing: 10           // 伏笔呼应 10%
  };

  // 综合评分
  calculateScore(checks: QualityChecks): QualityScore {
    const scores = {
      characterConsistency: this.checkCharacterConsistency(checks),
      plotLogic: this.checkPlotLogic(checks),
      worldSetting: this.checkWorldSetting(checks),
      timeline: this.checkTimeline(checks),
      foreshadowing: this.checkForeshadowing(checks)
    };

    // 加权计算总分
    const totalScore =
      scores.characterConsistency * this.dimensions.characterConsistency +
      scores.plotLogic * this.dimensions.plotLogic +
      scores.worldSetting * this.dimensions.worldSetting +
      scores.timeline * this.dimensions.timeline +
      scores.foreshadowing * this.dimensions.foreshadowing;

    return {
      total: Math.round(totalScore / 100),
      details: scores,
      issues: this.identifyIssues(scores),
      recommendation: this.getRecommendation(totalScore / 100)
    };
  }

  // 获取建议
  getRecommendation(score: number): string {
    if (score >= 90) return '优秀，可以直接发布';
    if (score >= 80) return '良好，可以继续';
    if (score >= 70) return '一般，建议优化';
    if (score >= 60) return '及格，需要修改';
    return '不及格，必须重写';
  }
}
```

**目标**：自动检测并修复常见问题，减少人工干预

|---------|---------|---------|
| **字数不足** | 字数统计 | 自动扩展内容 |
| **字数超标** | 字数统计 | 自动压缩内容 |
| **人物OOC** | 性格一致性检查 | 调整人物行为 |
| **时间线错误** | 时间线检查 | 修正时间顺序 |
| **设定冲突** | 设定一致性检查 | 调整设定描述 |

**自动修复流程**：

```typescript
class AutoFixer {
  // 自动修复入口
  async autoFix(chapter: string, issues: Issue[]): Promise<FixResult> {
    let fixedChapter = chapter;
    const fixLog: FixLog[] = [];

    for (const issue of issues) {
      try {
        const fix = await this.fixIssue(fixedChapter, issue);
        fixedChapter = fix.content;
        fixLog.push({
          type: issue.type,
          description: issue.description,
          action: fix.action,
          success: true
        });
      } catch (error) {
        fixLog.push({
          type: issue.type,
          description: issue.description,
          action: 'failed',
          success: false,
          error: error.message
        });
      }
    }

    return {
      content: fixedChapter,
      log: fixLog,
      success: fixLog.every(log => log.success)
    };
  }

  // 修复单个问题
  async fixIssue(chapter: string, issue: Issue): Promise<Fix> {
    switch (issue.type) {
      case 'word_count_insufficient':
        return this.fixWordCount(chapter, issue);
      case 'character_ooc':
        return this.fixCharacterOOC(chapter, issue);
      case 'timeline_error':
        return this.fixTimeline(chapter, issue);
      case 'setting_conflict':
        return this.fixSettingConflict(chapter, issue);
      case 'foreshadowing_missed':
        return this.fixForeshadowing(chapter, issue);
      default:
        throw new Error(`Unknown issue type: ${issue.type}`);
    }
  }

  // 修复字数不足
  async fixWordCount(chapter: string, issue: Issue): Promise<Fix> {
    const target = issue.targetWordCount || 2200;
    const controller = new WordCountController();
    const expanded = await controller.expandContent(chapter, target);

    return {
      content: expanded,
      action: `字数从${chapter.length}扩展到${expanded.length}`
    };
  }

  // 修复人物OOC
  async fixCharacterOOC(chapter: string, issue: Issue): Promise<Fix> {
    const character = issue.character;
    const profile = await this.getCharacterProfile(character);

    const prompt = `
    当前章节中，${character}的表现与设定不符。

    人物设定：
    - 性格：${profile.traits}
    - 背景：${profile.background}
    - 语言风格：${profile.speechStyle}

    问题段落：
    ${issue.problemParagraph}

    请修改这段内容，使人物表现符合设定。
    `;

    const fixed = await this.callAI(prompt);
    return {
      content: chapter.replace(issue.problemParagraph, fixed),
      action: `修正${character}的行为表现`
    };
  }
}
```

**功能概述**：

**技术架构**：

```typescript
// 模板定义
interface Template {
  id: string;
  name: string;
  description: string;
  category: 'genre' | 'structure' | 'scene' | 'chapter' | 'style';
  tags: string[];
  content: {
    structure: TemplateStructure;
    placeholders: Record<string, string>;
    examples: Record<string, string>;
  };
  rating: number;
  usage: number;
}

interface TemplateStructure {
  sections: TemplateSection[];
  metadata: Record<string, any>;
}

interface TemplateSection {
  title: string;
  description: string;
  required: boolean;
  placeholder: string;
  maxLength?: number;
}

// 模板管理器
class TemplateManager {
  private templates: Map<string, Template> = new Map();
  
  // 注册模板
  registerTemplate(template: Template) {
    this.templates.set(template.id, template);
  }
  
  // 获取模板
  getTemplate(templateId: string): Template | undefined {
    return this.templates.get(templateId);
  }
  
  // 列出模板
  listTemplates(category?: string, tags?: string[]): Template[] {
    let templates = Array.from(this.templates.values());
    
    if (category) {
      templates = templates.filter(t => t.category === category);
    }
    
    if (tags && tags.length > 0) {
      templates = templates.filter(t => 
        tags.some(tag => t.tags.includes(tag))
      );
    }
    
    return templates.sort((a, b) => b.rating - a.rating);
  }
  
  // 应用模板
  async applyTemplate(templateId: string, values: Record<string, string>): Promise<string> {
    const template = this.getTemplate(templateId);
    if (!template) {
      throw new Error(`Template ${templateId} not found`);
    }
    
    // 验证必需字段
    this.validateRequiredFields(template, values);
    
    // 填充模板
    const content = this.fillTemplate(template, values);
    
    return content;
  }
  
  // 评分模板
  rateTemplate(templateId: string, rating: number): void {
    const template = this.getTemplate(templateId);
    if (template) {
      template.rating = rating;
      template.usage++;
    }
  }
  
  // 搜索模板
  searchTemplates(query: string): Template[] {
    const all = Array.from(this.templates.values());
    return all.filter(t => 
      t.name.toLowerCase().includes(query.toLowerCase()) ||
      t.description.toLowerCase().includes(query.toLowerCase()) ||
      t.tags.some(tag => tag.toLowerCase().includes(query.toLowerCase()))
    );
  }
  
  private validateRequiredFields(template: Template, values: Record<string, string>): void {
    const required = template.content.structure
      .filter(s => s.required)
      .map(s => s.title);
    
    const missing = required.filter(r => !values[r]);
    
    if (missing.length > 0) {
      throw new Error(`Missing required fields: ${missing.join(', ')}`);
    }
  }
  
  private fillTemplate(template: Template, values: Record<string, string>): string {
    let content = '';
    
    for (const section of template.content.structure) {
      const value = values[section.title] || section.placeholder;
      content += `## ${section.title}\n\n${value}\n\n`;
    }
    
    return content;
  }
}
```

```typescript
  async analyzeRhythm(chapterContent: string): Promise<RhythmAnalysis> {

    const paragraphs = this.splitIntoParagraphs(chapterContent);
    
    // 分析句子长度变化
    const sentenceLengths = sentences.map(s => s.length);

    const paragraphVariance = this.calculateVariance(paragraphLengths);

    const issues = this.identifyRhythmIssues({
      sentenceVariance,
      paragraphVariance,
      emotionalIntensity,
      tension,
      pacingBreakdown
    });
    
    // 生成建议
    const suggestions = this.generateSuggestions(issues);
    
    // 确定整体节奏
    const overallPacing = this.determineOverallPacing(tension);
    
    return {
      overallPacing,
      tensionCurve,
      pacingBreakdown,
      suggestions,
      issues
    };
  }

    rhythmTable.tensionCurve = this.buildOverallTensionCurve(rhythmTable.chapters);
    
    // 生成调整建议
    rhythmTable.suggestedAdjustments = this.generateOverallSuggestions(rhythmTable);
    
    return rhythmTable;

          suggestions.push('缩短场景转换，保持紧张感');
        } else {
```

**功能概述**：
支持一个情节的多个发展方向，用户可以选择不同的剧情走向，保留所有版本。

```typescript
  id: string;
  parentId: string | null;
  chapterId: string;

// 分支管理器
class PlotBranchManager {
  private branches: Map<string, PlotBranch> = new Map();
  private branchTrees: Map<string, PlotBranchTree> = new Map();
  
  // 创建分支
  async createBranch(parentBranchId: string, newContent: string, style: string, metadata: any): Promise<PlotBranch> {
    const parentBranch = this.branches.get(parentBranchId);
    if (!parentBranch) {
      throw new Error(`Parent branch ${parentBranchId} not found`);
    }
    
    const newBranch: PlotBranch = {
      id: this.generateBranchId(),
      parentId: parentBranchId,
      chapterId: parentBranch.chapterId,
      content: newContent,
      style,
      createdAt: new Date(),
      updatedAt: new Date(),
      isSelected: false,
      metadata: {
        author: metadata.author || 'user',
        version: 1,
        tags: metadata.tags || []
      }
    };
    
    this.branches.set(newBranch.id, newBranch);
    
    // 更新分支树
    await this.updateBranchTree(parentBranch.chapterId, newBranch);
    
    return newBranch;
  }
  
  // 切换分支
  async switchBranch(branchId: string): Promise<void> {
    const branch = this.branches.get(branchId);
    if (!branch) {
      throw new Error(`Branch ${branchId} not found`);
    }
    
    // 取消当前选中的分支
    const tree = this.branchTrees.get(branch.chapterId);
    if (tree && tree.selectedBranch) {
      tree.selectedBranch.isSelected = false;
    }
    
    // 选中新分支
    branch.isSelected = true;
    
    if (tree) {
      tree.selectedBranch = branch;
    }
    
    // 更新章节内容
    await this.updateChapterContent(branch.chapterId, branch.content);
  }
  
  // 获取分支树
  async getBranchTree(chapterId: string): Promise<PlotBranchTree> {
    let tree = this.branchTrees.get(chapterId);
    
    if (!tree) {
      // 创建新的分支树
      const rootBranch = await this.findRootBranch(chapterId);
      tree = {
        root: rootBranch,
        branches: await this.getAllBranchesForChapter(chapterId),
        selectedBranch: rootBranch
      };
      this.branchTrees.set(chapterId, tree);
    }
    
    return tree;
  }
  
  // 合并分支
  async mergeBranches(branchIds: string[], mergeStrategy: 'latest' | 'vote' | 'custom'): Promise<string> {
    const branches = branchIds.map(id => this.branches.get(id)).filter(b => b !== undefined) as PlotBranch[];
    
    if (branches.length === 0) {
      throw new Error('No branches to merge');
    }
    
    let mergedContent: string;
    
    if (mergeStrategy === 'latest') {
      // 使用最新的分支
      const latest = branches.reduce((a, b) => a.updatedAt > b.updatedAt ? a : b);
      mergedContent = latest.content;
    } else if (mergeStrategy === 'vote') {
      // 使用AI投票选择最佳内容
      const agent = getAgent('review-agent');
      const result = await agent.execute(
        `请从以下${branches.length}个版本中选择最佳内容，并返回选择的内容：\n\n` +
        branches.map((b, i) => `版本${i + 1}：\n${b.content}`).join('\n\n')
      );
      mergedContent = result;
    } else if (mergeStrategy === 'custom') {
      // 自定义合并
      const agent = getAgent('writing-agent');
      const result = await agent.execute(
        `请合并以下${branches.length}个版本，保留各自的优点：\n\n` +
        branches.map((b, i) => `版本${i + 1}：\n${b.content}`).join('\n\n')
      );
      mergedContent = result;
    }
    
    return mergedContent;
  }
  
  // 删除分支
  async deleteBranch(branchId: string): Promise<void> {
    const branch = this.branches.get(branchId);
    if (!branch) {
      throw new Error(`Branch ${branchId} not found`);
    }
    
    if (branch.isSelected) {
      throw new Error('Cannot delete selected branch');
    }
    
    // 检查是否有子分支
    const hasChildren = Array.from(this.branches.values()).some(b => b.parentId === branchId);
    if (hasChildren) {
      throw new Error('Cannot delete branch with children');
    }
    
    this.branches.delete(branchId);
    
    // 更新分支树
    const tree = this.branchTrees.get(branch.chapterId);
    if (tree) {
      tree.branches = tree.branches.filter(b => b.id !== branchId);
    }
  }
  
  // 比较分支
  async compareBranches(branchId1: string, branchId2: string): Promise<BranchComparison> {
    const branch1 = this.branches.get(branchId1);
    const branch2 = this.branches.get(branchId2);
    
    if (!branch1 || !branch2) {
      throw new Error('One or both branches not found');
    }
    
    // 计算差异
    const diff = this.calculateDiff(branch1.content, branch2.content);
    
    // 分析风格差异
    const styleDiff = await this.analyzeStyleDifference(branch1, branch2);
    
    return {
      branch1: branch1.id,
      branch2: branch2.id,

    };
  }
  
  private generateBranchId(): string {
    return `branch-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;
  }
  
  private async updateBranchTree(chapterId: string, newBranch: PlotBranch): Promise<void> {
    let tree = this.branchTrees.get(chapterId);
    
    if (!tree) {
      const rootBranch = await this.findRootBranch(chapterId);
      tree = {
        root: rootBranch,
        branches: [rootBranch, newBranch],
        selectedBranch: rootBranch
      };
    } else {
      tree.branches.push(newBranch);
    }
    
    this.branchTrees.set(chapterId, tree);
```

```typescript
  private async analyzeWritingStyle(content: string): Promise<WritingStyle> {
    const sentences = this.splitIntoSentences(content);
    const paragraphs = this.splitIntoParagraphs(content);
    
    const sentenceLengths = sentences.map(s => s.length);
    const paragraphLengths = paragraphs.map(p => p.length);
    
    return {
      sentenceLength: {
        average: this.calculateAverage(sentenceLengths),
        variance: this.calculateVariance(sentenceLengths)
      },
      paragraphLength: {
        average: this.calculateAverage(paragraphLengths),
        variance: this.calculateVariance(paragraphLengths)
      },
      dialogueRatio: this.calculateDialogueRatio(content),
      descriptionRatio: this.calculateDescriptionRatio(content),
      showVsTell: this.calculateShowVsTell(content),
      tone: await this.analyzeTone(content),
      voice: await this.analyzeVoice(content)
    };

  private async identifyStructureType(chapters: ChapterStructure[]): Promise<'three-act' | 'hero-journey' | 'kishotenketsu' | 'custom'> {
    // 分析结构类型
    const climaxPosition = chapters.findIndex(c => c.plotFunction === 'climax');
    const climaxRatio = climaxPosition / chapters.length;
    
    if (climaxRatio > 0.7 && climaxRatio < 0.9) {
      return 'three-act';
    } else if (chapters.length >= 12) {
      return 'hero-journey';
    } else if (chapters.length === 4) {
      return 'kishotenketsu';
    } else {
      return 'custom';
    }
  }
```

### 6.3 提示词工程设计

#### 6.3.1 提示词工程核心原则

**基于 Claude 最佳实践**：

1. **明确指令**
   - 明确说明期望的输出格式
   - 避免模糊的描述
   - 使用清晰的分隔符（###、"""）

2. **提供上下文**
   - 解释为什么某些行为重要
   - 提供创作背景和动机
   - 帮助模型理解目标

3. **使用示例**
   - 提供正面和负面示例
   - 确保示例与期望行为一致
   - 最小化不期望的行为

4. **XML 结构化**
   - 使用 XML 标签组织提示词
   - 提高可读性和可维护性
   - 帮助模型理解结构

5. **控制输出格式**
   - 明确说明输出格式要求
   - 使用格式指示器
   - 匹配提示词风格到期望输出风格

### 6.3 小说创作核心理论

#### 6.3.1 故事结构理论

**1. 三幕式结构（Three-Act Structure）**

```
第一幕：设置（Setup，约25%）
- 激励事件（Inciting Incident）
- 第一转折点（Plot Point 1）
- 进入第二幕

第二幕：对抗（Confrontation，约50%）
- 上升动作（Rising Action）
- 中点（Midpoint）
- 第二转折点（Plot Point 2）
- 进入第三幕

第三幕：结局（Resolution，约25%）
- 高潮（Climax）
- 下降动作（Falling Action）
- 结局（Resolution）
```

**2. 英雄之旅（Hero's Journey，12个阶段）**

```
1. 平凡世界（Ordinary World）
2. 冒险召唤（Call to Adventure）
3. 拒绝召唤（Refusal of the Call）
4. 遇见导师（Meeting the Mentor）
5. 跨越第一道门槛（Crossing the First Threshold）
6. 试炼、盟友、敌人（Tests, Allies, Enemies）
7. 接近洞穴（Approach to the Inmost Cave）
8. 核心磨难（Ordeal）
9. 奖赏（Reward）
10. 回归之路（The Road Back）
11. 复活（Resurrection）
12. 带着灵药回归（Return with the Elixir）
```

**3. 救猫咪（Save the Cat，15个节拍）**

```
1. 开篇意象（Opening Image）
2. 主题呈现（Theme Stated）
3. 铺垫（Set-up）
4. 激励事件（Catalyst）
5. 争论（Debate）
6. 第二幕衔接点（Break into Two）
7. B故事（B Story）
8. 游戏时间（Fun and Games）
9. 中点（Midpoint）
10. 坏人逼近（Bad Guys Close In）
11. 一无所有（All Is Lost）
12. 灵魂黑夜（Dark Night of the Soul）
13. 第三幕衔接点（Break into Three）
14. 终局（Finale）
15. 终结意象（Final Image）
```

**4. 雪花法（Snowflake Method）**

```
步骤1：一句话概括故事
步骤2：一段话扩展（5个句子）
步骤3：人物表（姓名、目标、冲突、结局）
步骤4：扩展段落（每段变成一个章节）
步骤5：人物大纲（性格、故事线、背景）
步骤6：扩展章节大纲（每个段落）
步骤7：人物详细描述
步骤8：场景列表
步骤9：场景描写
步骤10：开始写作
```

#### 6.3.2 人物塑造理论

**1. 人物弧光（Character Arc）**

```
内在人物弧光（Internal Character Arc）：
- 初始状态：人物内心的缺陷或误解
- 激励事件：挑战人物的世界观
- 挣扎：人物试图保持旧方式
- 转折点：人物意识到需要改变
- 成长：人物开始改变
- 最终状态：人物完成转变

外在人物弧光（External Character Arc）：
- 初始状态：人物的外在目标
- 障碍：阻止人物达到目标
- 挣扎：人物克服障碍
- 高潮：人物达到或未达到目标
- 后果：目标的结果
```

**2. 人物原型（Character Archetypes）**

```
经典原型：
- 英雄（Hero）
- 导师（Mentor）
- 守门人（Threshold Guardian）
- 变形者（Shape-shifter）
- 影子（Shadow）
- 盟友（Ally）
- 诱骗者（Trickster）
```

**3. 人物动机层次（Maslow's Hierarchy）**

```
生理需求 → 安全需求 → 归属需求 → 尊重需求 → 自我实现
```

#### 6.3.3 叙事技巧理论

**1. 展示而非讲述（Show, Don't Tell）**

```
❌ 讲述：
她很生气。

✅ 展示：
她的拳头紧紧攥着，指节发白。呼吸急促，胸膛剧烈起伏。
```

**2. 感官细节（Sensory Details）**

```
五感描写：
- 视觉：颜色、形状、光线
- 听觉：声音、音乐、沉默
- 嗅觉：气味、香味、臭味
- 味觉：味道、质感
- 触觉：温度、质感、疼痛
```

**3. 对话技巧（Dialogue Techniques）**

```
对话原则：
- 推动情节
- 揭示人物
- 提供信息
- 制造冲突
- 节奏变化
- 避免信息堆砌
```

**4. 节奏控制（Pacing）**

```
节奏技巧：
- 短句创造紧张感
- 长句创造流畅感
- 段落长度影响节奏
- 场景切换控制节奏
- 高潮部分节奏加快
- 情感部分节奏放缓
```

```typescript
class SkillLoader {
  private skills: Map<string, NovelSkill> = new Map();
  
  async loadCommunitySkill(repo: string, path: string): Promise<NovelSkill> {
    // 从 GitHub 下载 Skill
    const content = await this.fetchFromGitHub(repo, path);
    
    // 解析 SKILL.md
    const skill = this.parseSkillMD(content);
    
    // 注册 Skill
    this.skills.set(skill.id, skill);
    
    return skill;
  }
  
  async loadLocalSkill(path: string): Promise<NovelSkill> {
    // 从本地文件系统加载
    const content = await fs.readFile(path, 'utf-8');
    const skill = this.parseSkillMD(content);
    
    this.skills.set(skill.id, skill);
    
    return skill;
  }
  
  getSkill(id: string): NovelSkill | undefined {
    return this.skills.get(id);
  }
  
  getSkillsByCategory(category: string): NovelSkill[] {
    return Array.from(this.skills.values())
      .filter(skill => skill.metadata.category === category);
  }
  
  getSkillsForAgent(agentId: string): NovelSkill[] {
    return Array.from(this.skills.values())
      .filter(skill => skill.metadata.compatibleAgents.includes(agentId));
  }
}
```

**5. Skill 与代理的集成**

```typescript
// 代理增强：添加 Skill 支持
class Agent {
  private skillLoader: SkillLoader;
  private availableSkills: NovelSkill[] = [];
  
  constructor(skillLoader: SkillLoader) {
    this.skillLoader = skillLoader;
  }
  
  async loadSkills() {
    // 加载兼容的 Skills
    this.availableSkills = this.skillLoader.getSkillsForAgent(this.id);
  }
  
  async executeWithSkill(task: string, skillId?: string) {
    let skill: NovelSkill | undefined;
    
    if (skillId) {
      // 使用指定的 Skill
      skill = this.skillLoader.getSkill(skillId);
    } else {
      // 自动选择合适的 Skill
      skill = this.selectBestSkill(task);
    }
    
    if (skill) {
      // 使用 Skill 增强执行
      return await this.executeWithSkillEnhanced(task, skill);
    } else {
      // 标准执行
      return await this.execute(task);
    }
  }
  
  private selectBestSkill(task: string): NovelSkill | undefined {
    // 根据任务描述选择最佳 Skill
    const taskLower = task.toLowerCase();
    
    for (const skill of this.availableSkills) {
      if (skill.config.autoActivate) {
        // 检查 Skill 描述是否匹配任务
        if (skill.description.toLowerCase().includes(taskLower) ||
            skill.metadata.tags.some(tag => taskLower.includes(tag.toLowerCase()))) {
          return skill;
        }
      }
    }
    
    return undefined;
  }
  
  private async executeWithSkillEnhanced(task: string, skill: NovelSkill) {
    // 使用 Skill 的系统提示词
    const enhancedSystemPrompt = skill.content.systemPrompt;
    
    // 加载 Skill 的资源
    const context = await this.loadSkillResources(skill);
    
    // 执行任务
    return await this.execute(task, {
      systemPrompt: enhancedSystemPrompt,
      context,
      templates: skill.content.templates
    });
  }
  
  private async loadSkillResources(skill: NovelSkill) {
    const resources: Record<string, string> = {};
    
    for (const [name, content] of Object.entries(skill.content.resources)) {
      // 如果是 URL，下载内容
      if (content.startsWith('http')) {
        resources[name] = await this.fetchURL(content);
      } else {
        // 直接使用内容
        resources[name] = content;
      }
    }
    
    return resources;
  }
}
```
