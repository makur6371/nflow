## 一、项目克隆和初始化

### 1.1 克隆项目

```bash

git clone https://github.com/iflow-ai/iflow-cli.git novel-flow
```


## 四、关键技术点

### 4.1 数据持久化

```typescript
// 使用 SQLite 存储小说数据
import Database from 'better-sqlite3';

const db = new Database('novel.db');

// 创建表
db.exec(NOVEL_SCHEMA);

// 数据访问层
class NovelRepository {
  saveCharacter(character: Character) {
    const stmt = db.prepare(`
      INSERT INTO characters (name, role, personality, background, traits, relationships)
      VALUES (?, ?, ?, ?, ?, ?)
    `);
    return stmt.run(character.name, character.role, character.personality, 
                    character.background, character.traits, character.relationships);
  }
  
  getCharacter(name: string): Character | undefined {
    const stmt = db.prepare('SELECT * FROM characters WHERE name = ?');
    return stmt.get(name);
  }
  
  getAllCharacters(): Character[] {
    const stmt = db.prepare('SELECT * FROM characters');
    return stmt.all();
  }
}
```

```typescript
// 继承 iflow 的记忆压缩机制
class NovelMemoryCompressor {
  compressCharacterMemory(characters: Character[]): CompressedMemory {
    // 保留关键信息，压缩细节
    const compressed = characters.map(char => ({
      name: char.name,
      role: char.role,
      keyTraits: this.extractKeyTraits(char.traits),
      recentActions: this.getRecentActions(char)
    }));
    
    return { type: 'character', data: compressed, compressedAt: Date.now() };
  }
  
  private extractKeyTraits(traits: string[]): string[] {
    // 提取最重要的性格特征
    return traits.slice(0, 5); // 只保留前5个
  }
}
```

## 六、常见问题和解决方案

### Q1: 如何处理 iflow 的依赖？

A: 
- 大部分依赖可以复用
- 只需要改造与代码相关的部分
- 新增小说创作特定的依赖

### Q2: 如何保持与 iflow 的兼容性？

A:
- 使用不同的命令名称（iflow vs nflow）
- 使用不同的配置目录（~/.iflow vs ~/.nflow）
- 可以同时安装和运行

### Q3: 如何处理版本更新？

A:
- fork iflow 项目后独立开发
- 定期合并 iflow 的更新
- 只合并核心架构部分，不合并具体实现
